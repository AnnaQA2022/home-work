1. Тест-кейс (Едадил (app)):

Название: Регистрация чека вручную.
Описание: Проверка регистрации чека при вводе данных вручную.
Предусловия: Зарегестрированный и авторизованный пользователь. Данные чека Шага 5 могут быть произвольные.
Эпик: Сканирование чека.
Время выполнения: 5 минут
Шаги:
1. Открыть приложение.
Открывается главная ЭФ приложения.
2. Перейти в "Сканер чеков".
Происходит переход на ЭФ Сканера чеков, на экран транслируется изображение с основной (тыловой) камеры телефона.
3. Нажать "Сканер чека".
Происходит переход в режим сканирования QR-кода.
4. Нажать "Ввести чек вручную".
Происходит переход на ЭФ Введения данных чека.
5. Ввести в форму данные чека: 
Дата и время покупки; 
Сумма чека; 
ФН (16 цифр);
ФД (3-5 цифр); 
ФП или ФПД (8-10 цифр).
6. Нажать "Отправить".
Происходит переход на главнуэ ЭФ приложения во вкладку "Кэшбек". Поверх ЭФ появляется popup с надписью "Чек успешно загружен".
7. Нажать "Понятно".


2. Баг-репорт:
ID: 001.
[Эпик]: формирование Списков в М.Видео (app).
Тема: Приложение закрывается при переходе в Списках из "Избранное" в "Сравнение".
Подробное описание: В разделе "Списки" при переходе из вкладки "Избранное" во вкладку "Сравнение" приложение закрывается, вкладка "Сравнение" недоступна.
Окружение: iPhone, iOS.
Шаги для воспроизведения:
1. запустить приложение;
2. добавить товары в "Избранное";
3. добавить товары в "Сравнение";
4. перейти в "Списки";
5. обновить "Избранное" с помощью скролла;
6. перейти во вкладку "Сравнение".
Фактический результат: Приложение закрывается.
Ожидаемый результат: Открывается вкладка "Сравнение" со списком товаров.
Приоритет: P2 Средний
Серьезность: S3 Значительный
Тестовая среда: Test Env
